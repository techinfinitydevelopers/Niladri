Setup Guide — AWS S3
1
Create AWS Account
Go to aws.amazon.com, create an account or sign in with an existing one. You'll need a valid email address, phone number, and a credit card (free tier is sufficient for initial setup).
2
Create S3 Bucket
Go to S3 → Create Bucket. Choose a unique globally-unique name (e.g. archive-lms-media). Select your preferred region. Uncheck "Block all public access" if you want to serve media files publicly without signed URLs. Acknowledge the warning and confirm creation.
3
Create IAM User
Go to IAM → Users → Add User. Give the user a name like archive-lms-s3-user. Under Permissions, choose "Attach policies directly" and attach the AmazonS3FullAccess policy (or a custom scoped policy for better security). After creation, go to the user's Security credentials tab and click Create access key. Download the credentials CSV — you won't be able to see the secret again.
4
Configure CORS
In your S3 bucket, go to Permissions → Cross-origin resource sharing (CORS) and paste the following JSON: 
[{"AllowedHeaders":["*"],"AllowedMethods":["GET","PUT","POST","DELETE"],"AllowedOrigins":["*"],"ExposeHeaders":[]}]
For production, replace "*" in AllowedOrigins with your actual domain.
5
Enter Keys Below
Paste your Access Key ID and Secret Access Key from the downloaded CSV into the configuration form below, fill in your bucket name and region, then click Test Connection to verify everything is working before saving.

Setup Guide — Email Delivery
1
Choose Email Provider
Popular SMTP options: Gmail (smtp.gmail.com:587), SendGrid (smtp.sendgrid.net:587), Mailgun (smtp.mailgun.org:587), and AWS SES (email-smtp.us-east-1.amazonaws.com:587). For production, SendGrid or AWS SES are recommended for deliverability.
2
Gmail App Password Setup
Enable 2FA on your Google account first. Then go to myaccount.google.com → Security → App Passwords. Click Generate password for "Mail" and your device. Use your Gmail address as the SMTP username and the generated 16-character password as the SMTP password. Note: regular Gmail passwords will not work.
3
SendGrid & Other Providers
For SendGrid: Log in → go to Settings → API Keys → Create API Key. Give it "Full Access" or "Mail Send" scope. Use the literal string apikey as the SMTP username and your generated API key as the SMTP password. For Mailgun: go to Sending → Domains and find your SMTP credentials under domain settings.
4
Test Your Configuration
After saving your configuration, use the Send Test Email button below to send a test message to your inbox. If the test fails, double-check your SMTP host, port, and credentials. Ensure your firewall allows outbound connections on port 587 or 465.

Setup Guide — Razorpay Payments
1
Create Razorpay Account
Go to razorpay.com/signup and create an account. Complete KYC verification to enable live payments. You can start with test mode without KYC.
2
Get API Keys
In your Razorpay Dashboard, go to Settings → API Keys → Generate Test Key. Copy the Key ID (starts with rzp_test_) and Key Secret. Paste them in the form below.
3
Configure Webhooks
In your Razorpay Dashboard, go to Settings → Webhooks → Add New. Set the webhook URL to: https://yourdomain.com/api/payments/webhook. Select events: payment.captured, payment.failed. Copy the Webhook Secret and paste it below.
4
Go Live
After testing all payment flows, complete KYC in your Razorpay Dashboard. Then generate Live Keys from Settings → API Keys → Generate Live Key. Toggle to Live Mode above and update the keys below.
